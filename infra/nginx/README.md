# `infra/nginx/` — Standalone nginx reverse-proxies for static doc sites

Two nginx servers run alongside the Spring Boot backend in the local compose
stack, each serving a different static documentation artifact. They are
**independent of the application** — they keep working when the backend is
restarting, under load, or crashed, because they only serve files from disk.

## Why standalone nginx, not `app://...maven-site` endpoints?

Spring Boot _can_ serve static resources (and does, via
`MavenSiteConfig`), but dedicating a tiny nginx container buys us:

1. **Independence** — the reports stay browsable even when the JVM is
   starting up (cold boot ~60 s) or crashed.
2. **Speed** — nginx serves static files 10–100× faster than a Java servlet
   container, with zero GC overhead.
3. **Lifecycle separation** — the reports are regenerated by a nightly CI
   schedule, not on every app boot, so they have a different cadence.
4. **No classpath bloat** — the report assets are big (Compodoc ships the
   whole TypeScript AST rendered as HTML). Keeping them out of the JAR
   keeps the image ~40 MB smaller.

## Files in this directory

| File               | Role                                                                                                                                                                  | Bound to                              | Port  |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------- | ----- |
| `compodoc.conf`    | Serves the Angular Compodoc site — API reference for all UI components, services, interfaces, and routes. Generated by `npm run compodoc` in the `mirador-ui` repo. | Container `/usr/share/nginx/html`     | 8085  |
| `maven-site.conf`  | Serves the Maven quality reports site — Surefire (unit tests), Failsafe (IT), JaCoCo (coverage), SpotBugs, Javadoc. Generated by `mvn site`.                          | Container `/usr/share/nginx/html`     | 8084  |
| `README.md`        | This file.                                                                                                                                                            | (not mounted)                         | —     |

## Mount wiring (excerpt from `docker-compose.yml`)

```yaml
maven-site:
  image: nginx:alpine
  ports: ["8084:80"]
  volumes:
    - ./infra/nginx/maven-site.conf:/etc/nginx/conf.d/default.conf:ro
    - ./target/site:/usr/share/nginx/html:ro

compodoc:
  image: nginx:alpine
  ports: ["8085:80"]
  volumes:
    - ./infra/nginx/compodoc.conf:/etc/nginx/conf.d/default.conf:ro
    - ../mirador-ui/docs/compodoc:/usr/share/nginx/html:ro
```

## Access

| URL                           | Content                                                  |
| ----------------------------- | -------------------------------------------------------- |
| <http://localhost:8084>       | Maven site: tests, coverage, SpotBugs, Javadoc            |
| <http://localhost:8085>       | Compodoc: Angular UI API reference                       |

Both are linked from the `/actuator/quality` backend endpoint so the Angular
dashboard's Quality page can open them in new tabs.

## When to edit these files

- **Change MIME types or caching rules**: edit the relevant `.conf`.
- **Expose under a sub-path**: modify `location /` and the compose volume mount.
- **Production**: these sites are NOT deployed to K8s. The reports are either
  generated in CI and published as pipeline artifacts, or served from a GCS
  bucket. See the GitLab CI `maven-site` job.
